# Gitignore Requirements

## Scope

Applies to repository ignore policy and generated local artifacts configured in `.gitignore` for Fountain.

R001  Statement: Exclude CMake/Ninja build outputs from version control.
Design: Ignore `build/`, `build-*`, and `cmake-build-*`.
Rationale: Build outputs are machine-generated and high-churn.
Tests:
- Run local configure/build and verify generated build artifacts do not appear in `git status`.

R005  Statement: Exclude editor and OS local metadata from version control.
Design: Ignore `.DS_Store`, `.vscode/`, `.idea/`, and temporary swap files.
Rationale: These files are developer-machine-local and non-portable.
Tests:
- Create representative local metadata files and verify they do not appear in `git status`.

R010  Statement: Keep source and requirements/test inputs tracked.
Constraints:
- Do not ignore `include/`, `src/`, `tests/`, `requirements/`, or `docs/`.
- Do not ignore `CMakeLists.txt`, shell scripts, or Bats files.
Tests:
- Verify tracked source/configuration files remain visible to git after ignore updates.

R015  Statement: Exclude local test/runtime scratch artifacts from version control.
Design: Ignore `tmp/`, `*.sqlite3`, and `*.db`.
Tests:
- Run local scripts/tests that produce temporary DB files and verify they do not appear in `git status`.
